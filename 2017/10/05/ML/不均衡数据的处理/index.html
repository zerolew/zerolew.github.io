不均衡数据的处理

发表于 2017-10-05 | 分类于 machine learning

本文旨在回顾论文Survey of resampling techniques for improving classification performance in unbalanced datasets。该论文中，Ajinkya More回顾了以往文献中提出的一些重采样技术，用其处理人工合成的不均衡数据集，并研究各种技术对分类性能的影响。论文中关于欠采样和过采样方法总结比较全面，算法描述清晰，实验配图丰富。论文中提及的算法见下图： 论文中关于CNN算法的描述，个人比较推荐看CNN for data reduction; 关于EasyEnsemble和BalanceCascade算法的描述，个人推荐看原始论文Exploratory Undersampling for Class-Imbalance Learning。

欠采样

多数类的随机欠采样

做法：在多数类中随机欠采样。（有放回、无放回抽样）
说明：多数类中每个点的近邻点为同类点时可以取得较好结果，类域交叉复杂的情况会丢失信息。

NearMiss系列

NearMiss系列方法根据多数类中点间距离进行欠采样。

  • NearMiss-1 -> 对多数类中的每个点，在少数类中找到它的k个近邻点，计算k个近邻点到该点的平均距离，保留多数类中前m个平均距离最小的点。
  • NearMiss-2 -> 对多数类中的每个点，在少数类中找到它的k个最远点，计算k个最远点到该点的平均距离，保留多数类中前m个平均距离最小的点。
  • NearMiss-3 -> 对少数类中的每个点，保留多数类中它的前k个近邻点。

CNN(Condensed Nearest Neighbor)

三类点：(1)Outliers，被其他类包围的点; (2)Prototypes，用于分类样本的最小点集; (3)Absorbed points，不属于(1)、(2)的点。
补充说明：

  • outliers产生原因：(1)随机错误; (2)该类样本不足; (3)其他类样本过多 (4)样本基于未知特征分类; (5)其他。
  • prototypes: 用于分类样本的最小点集合。对新样本点的类型判别，只需要该类型的点集。

CNN算法:

  1. 遍历训练集，按序移除每个点，同时检查该点能否被正确识别。
    • 能，放回。
    • 否，异常点，不放回。
  2. 对处理后的训练集进行排序，得到集合$X= \lbrace x_1,\dots,x_n \rbrace$，令$U=\lbrace x_1 \rbrace $，从$X$中移除$x_1$。
  3. 遍历$X$，对$X$中的点$x_i$，在$U$中找到它的最近邻点$y_i$，若$x_i$与$y_i$标签
    • 一致，$x_i$为absorbed points，无需加入$U$。
    • 不一致，$x_i$为prototypes，从$X$中移除，加入$U$。
  4. 重复step 3直到没有新的prototype加入。
    说明：
    • 链接中关于异常点的移除，个人理解是基于knn算法判别其是否能被正确判别。
    • 2中提及的排序方法如下：
      1. 对于训练集中的样本$x$，找到其不同类中的最近邻点$y$，再找到同类中距离$y$最近的点$x’$。
      2. 计算边界比例(border ratio) $\alpha(x)=||x’ - y||/||x -y||$，$||x’ - y||$和$||x -y||$为距离。
      3. 按照$\alpha(x)$值降序排列训练集得到$X$。
    • 2中排序方法倾向于将类的边界点加入$U$中。
    • 用$U$分类新样本，产生的结果和用原始数据集分类的结果可能略有不同。

ENN(Edited Nearest Neighbor)

做法：对多数类中的每个点，找到它的k个近邻点，若其所属类与其k个近邻点中同类标签占多数的类不一致，则移除该点。
扩展：Repeated Edited Nearest Neighbor，反复执行ENN，直到没有点可以移除。

Tomek Link Removal

Tomek Link： 一对点，若其互为最近邻点，且归属于不同的类，则这对点为Tomek Link。
做法：移除数据集中所有的Tomek Link。
扩展：移除Tomek Link中属于多数类的点。

One-Sided Selection

注：

  • 算法中的正样本为少数类，负样本为多数类
  • 一致子集：集合$C \subseteq S$，若使用集合$C$中的样本和1-NN规则能正确分类集合$S$中的样本，则$C$是$S$的一致子集。

过采样

随机过采样

做法：在少数类中随机过采样。(复制)
说明：易于过拟合。

SMOTE(Synthetic Minority Oversampling Technique)

算法：

  1. 对少数类中的每个点p，在少数类中找到它的k个近邻点。
  2. 选取r(r <= k)个近邻点，连接近邻点与点p构成r条连线。
  3. 在p与其近邻点的连线上随机选取一个点加入少数类中。

注：

  • SMOTE算法对少数类样本一视同仁，边界处样本容易错分。
  • 通常会将SMOTE算法和其他算法相结合(如SMOTE + Tomek Link Removal，SMOTE + ENN等)，以达到较好的效果。

Borderline-SMOTE

算法：
对于少数类中的每一个点p:

  1. 在训练集$T$中找到其m个近邻点构成集合$M_p$，令 $m’=|M_p \cap L|$
  2. 若$m’=m$，则点p是噪点，忽略点p并处理下一个点。
  3. 若$0 \leq m’ \leq \frac{m}{2}$，则点p安全，忽略点p并处理下一个点。
  4. 若$\frac{m}{2} \leq m’ \leq m$，则将点p放入DANGER集合。

两种形式：

  • Borderline-SMOTE1： 对DANGER集合中的点采用SMOTE算法生成新样本。
  • Borderline-SMOTE2： 对DANGER集合中的点，找到其最近邻点，最近邻点可以来自少数类，也可以来自多数类，在该点与最近邻点的连线上选取样本加入少数类。

Cluster-Based Oversampling

算法：

  1. 对多数类和少数类分别进行聚类(k-means)。
  2. 对多数类，通过随机过采样使得各簇样本数相同，都等于最大簇的样本数。
  3. 对少数类，通过随机过采样使得各簇样本数也相同，且总样本数与处理后的多数类的总样本数相同。

注：原始论文Class Imbalances versus Small Disjuncts中提到的几点比较有意思：

  • 论文认为small disjuncts才是导致不均衡数据分类性能降低的原因。
  • 数据不均衡不一定会导致分类性能降低；解决数据的不均衡也并非总能提升分类性能。
  • 分类器的偏置、属性噪声、缺失属性、类噪声和训练集大小使得small disjuncts容易错分。
