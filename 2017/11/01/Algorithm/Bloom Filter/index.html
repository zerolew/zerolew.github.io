Bloom Filter

发表于 2017-11-01 | 分类于 Algorithm

布隆过滤器是:

  • 一种概率型数据结构;
  • 用于检索元素是否属于某集合;
  • 优点:高空间效率、低错误率、高查询效率;
  • 缺点:存在误识别情况(false positive)、删除困难;

方法

说明

  2. $n$个元素的集合$S= {x_1, x_2, \dots, x_n}$
  3. $k$个相互独立的哈希函数$H = {h_1(x), h_2(x), \dots, h_k(x)}$

要点

错误率估计

假设：

  1. hash函数相互独立;
  2. hash函数等概率选择数组位置;

插入：

  1. 单次hash映射中，位数组某位为$0$(未选中)的概率: $$1 - \frac{1}{m}$$
  2. 插入一个元素，位数组某位为$0$(未选中)的概率: $$\left(1 - \frac{1}{m}\right)^k$$
  3. 插入整个集合，位数组某位为$0$(未选中)的概率: $$p = \left(1 - \frac{1}{m}\right)^{kn} \approx \mathrm{e}^{-kn/m} = p’$$

查询：

  1. 令$q$为位数组中$0$的比例，则$k$个hash函数对应位置全为1的概率是$(1 - q)^k$
  2. $q$的期望表示$n$个元素通过$k$个hash函数映射到位数组中，某个位置仍为$0$的概率，即$E[q] = (1 - 1/m)^{kn}$
  3. 查询某个元素，其hash函数对应的位置全为$1$的概率为: $$\left(1 - \left(1 - \frac{1}{m}\right)^{kn}\right)^k \approx \left(1 - \mathrm{e}^{-kn/m}\right)^k$$
  4. 误识别(false positive)率为$$ fr = \sum_t Pr(q = t)(1-t)^k \approx \left(1 - \left(1 - \frac{1}{m}\right)^{kn}\right)^k \approx \left(1 - \mathrm{e}^{-kn/m}\right)^k $$

说明：

  • $\left(1 - \frac{1}{m}\right)^{kn} \approx \mathrm{e}^{-kn/m}$用到公式$ \lim\limits_{x \to \infty} \left(1 - \frac{1}{x} \right)^{-x} = \mathrm{e} $

最优hash函数个数

假设$n$和$m$固定，最优化误识别率。

  • 取误识别率为$fr = \left(1 - \mathrm{e}^{-kn/m}\right)^k$

    1. 取误识别率的对数: $$f’ = k\ln(1 - \mathrm{e}^{-kn/m})$$
    2. 由$p’= \mathrm{e}^{-kn/m}$，得$$k = -\frac{m}{n}\ln{(p’)}$$
    3. $f’$变形为$$f’ = -\frac{m}{n}\ln{(p’)}\ln{(1-p’)}$$
    4. 对称性易知$p’=1/2$时，$f’$最小，此时$$k = \frac{m}{n}\ln{2}$$

  • 取误识别率为$fr = \left(1 - \left(1 - 1/m\right)^{kn}\right)^k$

    1. 取误识别率的对数: $$f = k \ln{\left(1 - \left(1 - 1/m\right)^{kn}\right)} $$
    2. 由$p = \left(1 - 1/m\right)^{kn}$，得$$k = \frac{\ln{(p)}}{n\ln\left(1 - 1/m \right)}$$
    3. $f$变形为$$f = \frac{\ln{(p)}\ln{(1-p)}}{n\ln\left(1 - 1/m \right)}$$
    4. 对称性易知$p=1/2$时，$f$最小，此时$$k = \frac{-\ln2}{n\ln{(1 - 1/m)}}$$

位数组大小

假设全集中共有$u$个元素，允许的最大错误率为$\epsilon$，且$X$为全集中任取$n$个元素的集合，$F(x)$是表示$X$的位数组。

  1. 位数组会误识别$\epsilon(u - n)$个元素(false positive)
  2. $F(x)$可以表示$\binom{n + \epsilon(u - n)}{n}$个集合
  3. $m$位的位数组共有$2^m$个不同的组合，则$m$位的位数组可以表示$2^m\binom{n + \epsilon(u - n)}{n}$个集合
  4. 全集中n个元素的集合共有$\binom{u}{n}$个

为了让$m$位的位数组能够表示所有$n$个元素的集合，必须满足$$2^m\binom{n + \epsilon(u - n)}{n} \ge \binom{u}{n}$$ 则有:$$m \ge \log_2 \frac{\binom{u}{n}}{\binom{n + \epsilon(u - n)}{n}} \approx \log_2 \frac{\binom{u}{n}}{\binom{\epsilon u}{n}} \ge \log_2 \epsilon^{-n} = n \log_2{(1/\epsilon)} $$

$k = (m\ln2)/n$时错误率$fr$取得最小值$fr = (1/2)^{m\ln2/n}$，令$fr \le \epsilon $则有:
$$ m \ge \frac{n \log_2(1/\epsilon)}{\ln2} = n \log_2 \mathrm{e} \cdot \log_2(1/\epsilon)$$

近似前提是$n$和$\epsilon u$相比很小，这也是实际情况中常常发生的。
结论:

  • 为了表示任意$n$个元素的集合，且使得$fr \le \epsilon$，必须满足$$m_{min} = n\log_2(1/\epsilon)$$
  • hash函数个数最优时，$ m \ge m_{min} \ln_2 \mathrm{e} $
